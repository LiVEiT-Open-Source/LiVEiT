# Standard LiVEiT® Project Structure

The official LiVEiT® HTML implementation follows a standard project layout.

Each project folder is named using the following format:

```text
YYYY_MM_DD_ProjectName_#Licence
```

Example:

```text
2026_06_28_gr0whealthcare_LiVEiT0003
```

A standard project contains:

```text
2026_06_28_gr0whealthcare_LiVEiT0003
│
├── index01.html
├── assets/
│   ├── hover_styles01.css
│   ├── *.svg
│   └── supporting framework assets
│
└── images/
    ├── background.jpg
    ├── hero.jpg
    └── project images
```

## Folder Purposes

### index01.html

The primary application page.

### assets/

Contains reusable framework resources including:

- CSS
- SVG graphics
- icons
- shared assets

### images/

Contains project-specific images such as:

- backgrounds
- logos
- hero graphics
- client images

All official LiVEiT® HTML projects should follow this structure to maintain consistency across implementations.
