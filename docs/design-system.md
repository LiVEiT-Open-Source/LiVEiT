# LiVEiT® Design System

## Introduction

The LiVEiT® Design System defines the visual language of the LiVEiT® Framework.

It provides guidance for developers, designers and organisations wishing to create implementations that remain recognisably LiVEiT® while allowing flexibility for individual projects.

The goal is consistency rather than restriction.

---

# Design Philosophy

The LiVEiT® Framework has been designed around several core principles.

- Clarity
- Simplicity
- Accessibility
- Consistency
- Flexibility
- Collaboration
- Understanding

Every visual decision should support these principles.

---

# The Circle

The circle is the primary visual component of the LiVEiT® Framework.

Rather than presenting information as a sequence of steps, the circle represents continuous exploration, reflection and learning.

There is no defined beginning or end.

Users are encouraged to approach the framework from whichever position best suits their current needs.

---

# The Twelve Positions

Each position within the circle represents a framework principle rather than a fixed topic.

Implementations may adapt the meaning of each position while preserving the structure of the framework.

This separation between structure and implementation is fundamental to LiVEiT®.

---

# Colour Philosophy

Colours are used to support recognition rather than decoration.

Each position should maintain a consistent colour identity wherever practical.

The official implementation currently uses:

| Position | Colour |
|----------|---------|
| 1 | Blue |
| 2 | Orange |
| 3 | Green |
| 4 | Brown |
| 5 | Grey |
| 6 | White |
| 7 | Red |
| 8 | Black |
| 9 | Yellow |
| 10 | Violet |
| 11 | Rose |
| 12 | Aqua |

Alternative palettes may be used where appropriate, provided accessibility is maintained.

---

# Visual Hierarchy

Information should always be presented clearly.

Recommended hierarchy:

1. Circle
2. Content
3. Navigation
4. Supporting information

Visual effects should never dominate the content.

---

# Glass Design

Many official LiVEiT® implementations use a glass inspired interface.

Characteristics include:

- Subtle transparency
- Soft gradients
- Gentle highlights
- Light blur
- Rounded corners
- Minimal borders

Glass effects should improve readability rather than reduce it.

---

# Motion

Animation should communicate state rather than provide decoration.

Recommended uses include:

- Hover feedback
- Focus changes
- Modal transitions
- Navigation
- Loading states

Animations should remain smooth, subtle and optional where possible.

---

# Typography

Typography should prioritise readability.

Guidelines include:

- Clear headings
- Consistent spacing
- Accessible sizing
- Strong hierarchy
- Comfortable line length

Typography should support understanding.

---

# Icons

Icons should support navigation rather than replace text.

Where possible, icons should be accompanied by labels to improve accessibility.

---

# Responsive Design

The framework should adapt naturally to different screen sizes.

Implementations should support:

- Desktop
- Laptop
- Tablet
- Mobile

No functionality should depend upon screen size.

---

# Accessibility

Accessibility is one of the core principles of the LiVEiT® Framework.

Every implementation should support:

- Keyboard navigation
- Screen readers
- High contrast
- Reduced motion
- Responsive layouts
- Clear focus indicators

Accessibility should always be considered during the design process rather than added afterwards.

---

# Branding

The LiVEiT® Framework is open source.

The LiVEiT® name, logo and branding remain protected by trademark.

Please refer to TRADEMARK.md for guidance on acceptable use.

---

# Design Consistency

Developers are encouraged to innovate.

However, implementations should remain recognisably based upon the LiVEiT® Framework.

Consistency helps users move between implementations with confidence and familiarity.

---

# Looking Forward

The Design System will continue to evolve alongside the framework.

As the community grows, new recommendations and best practices will be incorporated while preserving the core identity of LiVEiT®.

The objective is not to create identical implementations.

The objective is to create compatible implementations built upon shared principles.
