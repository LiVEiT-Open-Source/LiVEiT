# CSS Guide

## Introduction

The LiVEiT® Framework uses CSS to create a clean, accessible and responsive user experience.

The objective is not simply to make the framework attractive, but to ensure that visual design supports understanding, interaction and accessibility.

---

# Design Philosophy

The visual design of LiVEiT® follows several guiding principles:

- Simplicity over complexity.
- Clarity over decoration.
- Accessibility over novelty.
- Consistency across implementations.
- Responsive by default.

Visual elements should always support the purpose of the framework rather than distract from it.

---

# Colour System

Each position within the LiVEiT® Circle is represented by a distinct colour.

Colours should remain consistent wherever practical to help users build familiarity with the framework.

Developers may adapt colour palettes where appropriate, provided accessibility and sufficient colour contrast are maintained.

---

# Responsive Design

Implementations should adapt naturally to different screen sizes.

Layouts should be designed to work across:

- Desktop computers
- Laptops
- Tablets
- Mobile phones

The framework should remain usable regardless of screen size.

---

# Typography

Typography should prioritise readability.

Recommended principles include:

- Clear headings
- Consistent spacing
- Accessible font sizes
- Logical hierarchy
- Sufficient line spacing

Typography should support understanding rather than visual complexity.

---

# Animation

Animation should be subtle and purposeful.

Recommended uses include:

- Hover feedback
- Focus indicators
- Modal transitions
- State changes

Animations should never interfere with usability or accessibility.

---

# Glass Effects

Many official LiVEiT® implementations use a glass inspired interface.

This visual style provides depth while allowing content to remain the primary focus.

Developers are free to adopt alternative visual styles provided the underlying framework remains recognisable.

---

# Accessibility

Every CSS implementation should consider:

- High contrast
- Colour contrast
- Keyboard focus indicators
- Reduced motion preferences
- Responsive scaling

Accessibility should always take priority over aesthetics.
